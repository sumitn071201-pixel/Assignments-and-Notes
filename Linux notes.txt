## 1. `case-sac statement.txt`&#x20;

This is a **bash `case` statement** example.

```bash
Name="Girish"

case "$Name" in
    #case 1
    "Rajib") echo "Profession : Software Engineer" ;;
    
    #case 2
    "Vikas") echo "Profession : Web Developer" ;;
    
    #case 3
    "Girish") echo "Profession : Technical Content Writer" ;;
esac
```

### How it works:

* `Name="Girish"` â†’ variable `Name` holds the value `"Girish"`.
* `case "$Name" in ... esac` â†’ checks the value of `Name` against patterns.
* `"Rajib") ... ;;` â†’ if `Name` is `"Rajib"`, it prints `"Profession : Software Engineer"`.
* `"Vikas") ... ;;` â†’ if `Name` is `"Vikas"`, it prints `"Profession : Web Developer"`.
* `"Girish") ... ;;` â†’ if `Name` is `"Girish"`, it prints `"Profession : Technical Content Writer"`.

ğŸ‘‰ In this script, since `Name="Girish"`, the output will be:

```
Profession : Technical Content Writer
```

So basically:

* `case` works like a **switch-case** in C/Java.
* `;;` means "end of this case".
* `esac` is just `case` spelled backwards â†’ used to close the block.

## 2. `Command line arguments.txt`&#x20;

This is about **special variables** in bash used for handling arguments.

### Example script:

```bash
#!/bin/bash

echo "Script name is $0"
echo "The 2nd argument passed is: $2"
echo "Arguments passed to script are: $*"
```

### Explanation of special variables:

* `$0` â†’ name of the script itself.
* `$1`, `$2`, ... â†’ first, second, etc. arguments.
* `$#` â†’ number of arguments passed.
* `$*` â†’ all arguments as a single string.
* `$@` â†’ all arguments but quoted separately.
* `$?` â†’ exit status of last command (0 = success, non-0 = error).
* `$_` â†’ last argument of previous command.
* `$$` â†’ process ID (PID) of the current shell.
* `$-` â†’ flags/options set in the shell.

### ğŸ”¹ **Until Loop**


* Opposite of `while`.
